üß© Chapter 1: What is NGINX?

Once upon a time, web servers like Apache were the default ‚Äî they handled everything: serving pages, routing, even running scripts. But as traffic grew, they struggled.

Then came NGINX (pronounced "engine-x") ‚Äî built for speed, scalability, and simplicity.

Think of NGINX as:

  • A traffic cop that directs incoming requests
  • A smart waiter who delivers web pages to users
  • A bouncer who decides who gets in
In short: NGINX = reverse proxy + web server + load balancer + caching engine ‚Äî all in one lightweight tool.

Let's understand the important pieces before setup üëá

üß± 1. Server Block

A server block (like a chapter in a config file) tells NGINX:

"Hey, when someone visits this domain or IP, serve them like this."

server {
  listen 80;
  server_name example.com;
  root /var/www/example;
  index index.html;
}

üß≠ 2. Location Block

Inside a server block, you tell NGINX what to do for specific paths:

location /api {
  proxy_pass http://localhost:8080;
}

Means ‚Üí If a request comes to /api, forward it to another backend (like your Go app).

üîÅ 3. Reverse Proxy

When NGINX sits in front of your backend and forwards requests, it's acting as a reverse proxy.

Example: Browser ‚Üí NGINX (port 80) ‚Üí Go API (port 8080)

This gives you:

  • Security (hide backend IP)
  • Load balancing
  • Centralized logging and rate limiting

‚ö° 4. Load Balancer

NGINX can split traffic between multiple backend servers:

upstream backend {
  server backend1.example.com;
  server backend2.example.com;
}

server {
  location / {
    proxy_pass http://backend;
  }
}

NGINX automatically rotates requests across backends.

üîí 5. SSL/TLS (HTTPS)

To secure your site:

Use listen 443 ssl; and add certificate and key paths.

server {
  listen 443 ssl;
  ssl_certificate /etc/nginx/ssl/mycert.crt;
  ssl_certificate_key /etc/nginx/ssl/mykey.key;
}

üèóÔ∏è Chapter 5: Example ‚Äî Host Your Go App via NGINX

Suppose your Go API runs on localhost:8080.

Create file:

sudo nano /etc/nginx/sites-available/goapp
Paste this configuration:
server {
  listen 80;
  server_name myapp.local;

  location / {
    proxy_pass http://localhost:8080;
    proxy_set_header Host $host;
    proxy_set_header X-Real-IP $remote_addr;
  }
}
Enable site:
sudo ln -s /etc/nginx/sites-available/goapp /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx

Now visit ‚Üí http://myapp.local

Boom üí• You're seeing your Go app through NGINX!

üõ°Ô∏è Chapter 7: Hardening & Optimization

Enable Gzip Compression

gzip on;
gzip_types text/plain text/css application/json application/javascript;

Cache static assets

location ~* \.(jpg|jpeg|png|gif|ico|css|js)$ {
  expires 7d;
  add_header Cache-Control "public";
}

Limit requests (DDoS protection)

limit_req_zone $binary_remote_addr zone=one:10m rate=1r/s;

server {
  location /login/ {
    limit_req zone=one burst=5 nodelay;
  }
}
